## 4. 创建虚拟环境 & 安装依赖

```bash
cd /inspire/hdd/project/wuliqifa/yanjunchi-24040/why/QIHC
source experiments/nsfc_evidence/cn_mirror_env.sh

python3 -m venv .venv
source .venv/bin/activate

pip install -U pip -i "$PIP_INDEX_URL" --trusted-host pypi.tuna.tsinghua.edu.cn
pip install -i "$PIP_INDEX_URL" --trusted-host pypi.tuna.tsinghua.edu.cn -e ".[dev,hf,llm]"
pip install -i "$PIP_INDEX_URL" --trusted-host pypi.tuna.tsinghua.edu.cn modelscope

python -c "import torch; print('CUDA:', torch.cuda.is_available(), 'nGPU:', torch.cuda.device_count())"
```

**必须看到 `CUDA: True` 且 `nGPU: 2`。** 若为 `CUDA: False`，先完成 **§4.1** 再往下走。

### 4.1 PyTorch 与驱动不匹配（`CUDA: False` / driver too old）

**典型报错：**

```
UserWarning: CUDA initialization: The NVIDIA driver on your system is too old
(found version 12040). ...
CUDA: False nGPU:
```

**原因**：`pip install` 装到的 PyTorch 是用**较新 CUDA** 编译的，而节点驱动只支持较低 CUDA（如 12.4 → `12040`）。不是 4090 坏了，是 **torch 轮子与驱动不匹配**。

**Step A — 查看驱动支持的 CUDA 上限：**

```bash
nvidia-smi
```

记下右上角：

```
Driver Version: 535.xx
CUDA Version: 12.x          ← 选 PyTorch 轮子依据这一行
```

**Step B — 卸载并重装匹配版本的 torch：**

```bash
source .venv/bin/activate
source experiments/nsfc_evidence/cn_mirror_env.sh

pip uninstall -y torch torchvision torchaudio

# 优先：驱动 CUDA Version >= 12.4 或报错含 12040
pip install torch torchvision torchaudio \
  -i https://pypi.tuna.tsinghua.edu.cn/simple \
  --extra-index-url https://download.pytorch.org/whl/cu124

# 若仍 CUDA: False，改试 cu121（CUDA Version 12.1 / 12.2 常见）
# pip install torch torchvision torchaudio \
#   -i https://pypi.tuna.tsinghua.edu.cn/simple \
#   --extra-index-url https://download.pytorch.org/whl/cu121

# 更老驱动（CUDA Version 11.x）
# pip install torch torchvision torchaudio \
#   -i https://pypi.tuna.tsinghua.edu.cn/simple \
#   --extra-index-url https://download.pytorch.org/whl/cu118
```

清华 PyTorch 镜像说明：<https://mirrors.tuna.tsinghua.edu.cn/help/pytorch/>

**Step C — 验证：**

```bash
python -c "
import torch
print('CUDA:', torch.cuda.is_available())
print('nGPU:', torch.cuda.device_count())
print('torch:', torch.__version__)
print('built_cuda:', torch.version.cuda)
"
python -c "import torch; print(torch.zeros(1, device='cuda:0'))"
```

期望输出含 `CUDA: True`、`nGPU: 2`、`GPU0 OK`。

**Step D — 仍失败时：**

| 情况 | 处理 |
|------|------|
| 集群不能升级驱动 | 换 CUDA 更低的 cu118/cu121 轮子，或换 GPU 节点 |
| 装成 CPU 版 torch | 必须带 `--extra-index-url https://download.pytorch.org/whl/cu12x` |
| 无权限改驱动 | 联系 Inspire 管理员 |

CUDA 正常后再执行第 5 节（下模型）及第 6 节（跑实验）。

## 5. 下载 7B 模型（ModelScope，国内推荐）

> HF 镜像在 `Fetching 14 files: 0%` 停很久是常见现象（首文件 ~15GB）。**国内请用 ModelScope。**

### 5.1 单独下载（推荐先跑通）

```bash
cd /inspire/hdd/project/wuliqifa/yanjunchi-24040/why/QIHC
source .venv/bin/activate
source experiments/nsfc_evidence/cn_mirror_env.sh

mkdir -p offline_bundle/models/Qwen2.5-7B-Instruct

python experiments/nsfc_evidence/download_model_hf.py \
  --repo Qwen/Qwen2.5-7B-Instruct \
  --local-dir offline_bundle/models/Qwen2.5-7B-Instruct \
  --backend modelscope
```

成功标志：

```bash
ls offline_bundle/models/Qwen2.5-7B-Instruct/config.json
# 文件存在即可
```

### 5.2 ModelScope CLI 备选

```bash
pip install modelscope
modelscope download --model Qwen/Qwen2.5-7B-Instruct \
  --local_dir offline_bundle/models/Qwen2.5-7B-Instruct
```

### 5.3 预下载 BBH 数据集

```bash
python experiments/download_bbh_hf.py --limit-per-task 50
```

## 6. 运行全量实验

### 6.1 模型已下好 → 跳过下载（推荐）

```bash
cd /inspire/hdd/project/wuliqifa/yanjunchi-24040/why/QIHC
source .venv/bin/activate
export CUDA_VISIBLE_DEVICES=0,1

SKIP_MODEL_DOWNLOAD=1 bash experiments/nsfc_evidence/run_dual_4090.sh
```

### 6.2 一键从头跑（含自动下载）

```bash
CUDA_VISIBLE_DEVICES=0,1 bash experiments/nsfc_evidence/run_dual_4090.sh
```

### 6.3 后台运行（推荐，约 12–24 小时）

```bash
cd /inspire/hdd/project/wuliqifa/yanjunchi-24040/why/QIHC
source .venv/bin/activate
export CUDA_VISIBLE_DEVICES=0,1

nohup bash experiments/nsfc_evidence/run_dual_4090.sh \
  > experiments/outputs/nsfc_evidence/dual_4090_launcher.log 2>&1 &

echo $!
tail -f experiments/outputs/nsfc_evidence/dual_4090_launcher.log
```

### 6.4 赶时间（减小规模）

```bash
HF_LIMIT_PER_TASK=25 SEEDS="0 1 2" SKIP_MODEL_DOWNLOAD=1 \
  bash experiments/nsfc_evidence/run_dual_4090.sh
```

### 6.5 环境已装好

```bash
SKIP_SETUP=1 SKIP_MODEL_DOWNLOAD=1 bash experiments/nsfc_evidence/run_dual_4090.sh
```

## 10. 故障排除

### pip 连不上 PyPI

```bash
source experiments/nsfc_evidence/cn_mirror_env.sh
pip install -i "$PIP_INDEX_URL" --trusted-host pypi.tuna.tsinghua.edu.cn <package>
```

### HF `401 Unauthorized` / `cas-server.xethub.hf.co`

```bash
export HF_HUB_DISABLE_XET=1
export HF_ENDPOINT=https://hf-mirror.com
pip uninstall -y hf-xet   # 可选
```

改用 ModelScope（见第 5 节）。

### HF 下载卡在 `Fetching 14 files: 0%`

1. `Ctrl+C` 停止  
2. 用 **ModelScope** 下载（第 5.1 节）  
3. `SKIP_MODEL_DOWNLOAD=1` 再跑实验  

不是死机：首文件约 15GB，HF 镜像可能极慢。

### PyTorch `CUDA: False` / `driver too old (found version 12040)`

完整步骤见 **§4.1**。快速修复：

```bash
pip uninstall -y torch torchvision torchaudio
pip install torch torchvision torchaudio \
  -i https://pypi.tuna.tsinghua.edu.cn/simple \
  --extra-index-url https://download.pytorch.org/whl/cu124
python -c "import torch; print(torch.cuda.is_available(), torch.cuda.device_count())"
```

### OOM（显存不足）

- 确保 `--model-name` 指向本地 `offline_bundle/models/...`  
- 减小规模：`HF_LIMIT_PER_TASK=25`  
- 代码内将 `LLMSamplerConfig.batch_size` 从 8 改为 4  

### 中断后续跑

模型已下好：

```bash
SKIP_SETUP=1 SKIP_MODEL_DOWNLOAD=1 bash experiments/nsfc_evidence/run_dual_4090.sh
```

单个子任务失败：查看 `${RUN}/w*.log`，复制其中 Python 命令单独重跑。
