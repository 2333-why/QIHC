# Pro 6000 · 单卡约 2 小时快速实验

> **场景**：`pro6000` 只有 **1 张空闲 GPU**，先跑小规模验证流水线，再决定是否开全量。  
> **脚本**：`experiments/nsfc_evidence/run_pro6000_quick_2h.sh`

---

## 1. 会跑什么（单卡串行）

| 阶段 | 内容 | 预计耗时 |
|------|------|----------|
| **A** | bundled **mock** 五模式（无 LLM） | ~3 min |
| **B** | bundled **10 题 × 7B** CR 五模式（主表） | ~60–90 min |
| **C** | handoff **10 题** 7B | ~20–40 min |
| **D** | Case F 消融(50题 mock) + sampler + F 曲线 | ~10 min |

**合计约 1.5–2.5 小时**（视 GPU 与磁盘而定）。

全量脚本 `run_dual_pro6000.sh` 用 50 采样、40+ 题、双卡，需十数小时。

## 2. 前置条件

```bash
cd /hdd/why/QIHC
source .venv/bin/activate

# CUDA 必须 True（见双4090手册 §4.1）
python -c "import torch; print(torch.cuda.is_available(), torch.cuda.device_count())"

# 7B 模型建议已下到本地
ls offline_bundle/models/Qwen2.5-7B-Instruct/config.json
```

---

## 3. 一键运行

```bash
cd /hdd/why/QIHC
chmod +x experiments/nsfc_evidence/run_pro6000_quick_2h.sh

# 指定空闲 GPU（例如只有 GPU 1 空闲）
export CUDA_VISIBLE_DEVICES=1

# 模型已下好、环境已装好：
export SKIP_SETUP=1
export SKIP_MODEL_DOWNLOAD=1

bash experiments/nsfc_evidence/run_pro6000_quick_2h.sh
```

**后台：**

```bash
nohup bash experiments/nsfc_evidence/run_pro6000_quick_2h.sh \
  > experiments/outputs/nsfc_evidence/pro6000_quick_launcher.log 2>&1 &
tail -f experiments/outputs/nsfc_evidence/pro6000_quick_launcher.log
```

---

## 4. 调参（更短 / 更长）

```bash
# 更短（约 1h）：5 题、6 次采样
QUICK_N_TASKS=5 QUICK_N_SAMPLES=6 QUICK_HANDOFF_LIMIT=5 \
  SKIP_SETUP=1 SKIP_MODEL_DOWNLOAD=1 \
  bash experiments/nsfc_evidence/run_pro6000_quick_2h.sh

# 稍长（约 3h）：15 题、12 次采样
QUICK_N_TASKS=15 QUICK_N_SAMPLES=12 QUICK_HANDOFF_LIMIT=15 \
  bash experiments/nsfc_evidence/run_pro6000_quick_2h.sh
```

| 变量 | 默认 | 说明 |
|------|------|------|
| `CUDA_VISIBLE_DEVICES` | `0` | 空闲 GPU 编号 |
| `QUICK_N_TASKS` | `10` | B 阶段 bundled 题数 |
| `QUICK_N_SAMPLES` | `8` | CR 采样（全量 50） |
| `QUICK_BUDGET` | `300` | p-bit 步数 |
| `QUICK_HANDOFF_LIMIT` | `10` | handoff 题数 |
| `SKIP_SETUP` | `0` | `1`=跳过 pip/下模型 |
