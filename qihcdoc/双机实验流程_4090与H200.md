# QIHC 双机实验完整流程：联网 4090 → 离线 H200

> 适用场景：一张 **联网 RTX 4090** 负责下载与轻量验证；四张 **不联网 H200** 负责全量批量实验。  
> 对应脚本：`experiments/nsfc_evidence/run_online_4090.sh`、`experiments/nsfc_evidence/run_offline_h200.sh`  
> 仓库：<https://github.com/2333-why/QIHC>

---

## 分工总览

| 机器 | 网络 | 脚本 | 主要职责 |
|------|------|------|----------|
| **RTX 4090** | 联网 | `run_online_4090.sh` | 下载模型/BBH、打包 `offline_bundle/`、7B 轻量 P0 实验 |
| **4× H200** | 离线 | `run_offline_h200.sh` | 读 `manifest.env`，4 卡分波并行跑全量实验 |

```
4090（联网）                    H200（离线）
─────────────                  ─────────────
git pull                       接收 offline_bundle/
run_online_4090.sh      ──►    run_offline_h200.sh
offline_bundle/                全量结果 tar 拷回
轻量 7B 实验（可先写申请书）
```

---

## 阶段 0：本机 / 仓库准备（一次性）

在开发机（Mac）或 4090 上确保代码为最新版：

```bash
cd /path/to/QIHC
git pull origin main
chmod +x experiments/nsfc_evidence/run_online_4090.sh
chmod +x experiments/nsfc_evidence/run_offline_h200.sh
```

若 4090 尚未有仓库：

```bash
git clone https://github.com/2333-why/QIHC.git
cd QIHC
```

## 阶段 1：联网 4090（按顺序执行）

### Step 1.1 环境检查

```bash
cd /path/to/QIHC

nvidia-smi                # 确认 4090 可见
python3 --version         # 需要 >= 3.10
```

### Step 1.2 运行联网流水线（核心一步）

```bash
CUDA_VISIBLE_DEVICES=0 bash experiments/nsfc_evidence/run_online_4090.sh
```

**自动完成的内容：**

| 序号 | 动作 | 产出路径 |
|------|------|----------|
| 1 | 创建 `.venv` 并安装依赖 | `.venv/` |
| 2 | 下载 Qwen2.5-7B | `offline_bundle/models/Qwen2.5-7B-Instruct/` |
| 3 | 下载 Qwen2.5-14B（默认开启） | `offline_bundle/models/Qwen2.5-14B-Instruct/` |
| 4 | 缓存 BBH 数据集 | `offline_bundle/hf_home/` |
| 5 | 打包 pip wheels | `offline_bundle/wheels/` |
| 6 | 生成离线 manifest | `offline_bundle/manifest.env` |
| 7 | 4090 轻量实验 | `experiments/outputs/nsfc_evidence/online_4090_*/` |

**预计耗时**：下载 1–3 小时（视网速）；轻量实验 2–6 小时。

### Step 1.3 常用选项

```bash
# 只下载打包，不跑实验（赶时间先传 H200）
SKIP_EXPERIMENTS=1 bash experiments/nsfc_evidence/run_online_4090.sh

# 不下载 14B（省约 30GB；H200 上需 SKIP_CASE_H=1）
DOWNLOAD_14B=0 bash experiments/nsfc_evidence/run_online_4090.sh

# 跳过 wheels 打包
SKIP_WHEELS=1 bash experiments/nsfc_evidence/run_online_4090.sh
```

### Step 1.4 检查离线包是否齐全

```bash
ls offline_bundle/manifest.env
ls offline_bundle/models/Qwen2.5-7B-Instruct/config.json
ls offline_bundle/hf_home/
ls offline_bundle/wheels/ | head
```

以上四项齐全后，方可拷到 H200。

### Step 1.5 查看 4090 轻量实验产出（可先写申请书草稿）

```bash
RUN=$(ls -td experiments/outputs/nsfc_evidence/online_4090_* | head -1)
ls "$RUN"

# 重点文件：
#   unified_bundled_7b/unified_ablation.json   — bundled 7B 主表
#   handoff_7b/handoff.json                    — handoff 曲线
#   cr_hf_bbh_25/cr_protocol.json              — HF CR 子集
#   pareto_mock/                               — Pareto mock
```

### Step 1.6 打包便于传输

脚本结束时会生成 `offline_bundle/qihc_offline_bundle_*.tar.gz`。也可手动打包：

```bash
cd /path/to/QIHC

# 仅离线包（H200 上已有代码时）
tar -czf ~/offline_bundle_only.tar.gz offline_bundle/

# 代码 + 离线包（排除 venv）
tar -czf ~/qihc_for_h200.tar.gz \
  --exclude='.venv' \
  --exclude='__pycache__' \
  --exclude='.git' \
  .
```

## 阶段 2：数据传输（4090 → H200）

H200 **不能访问外网**，必须物理拷贝 `offline_bundle/`。

### 必须拷贝的内容

```
QIHC/
└── offline_bundle/              ← 必须
    ├── manifest.env
    ├── models/
    │   ├── Qwen2.5-7B-Instruct/
    │   └── Qwen2.5-14B-Instruct/   （若 DOWNLOAD_14B=0 则无）
    ├── hf_home/
    └── wheels/
```

若 H200 上已有 QIHC 代码（`git pull` 或旧版），**至少拷贝整个 `offline_bundle/`**。

### 传输方式（任选）

```bash
# U 盘：直接复制目录或 tar 包

# 内网 scp（4090 → H200，替换 IP 与用户）
scp -r /path/to/QIHC/offline_bundle  wl@<H200_IP>:/hdd/why/QIHC/

# 或传整个 tar
scp ~/offline_bundle_only.tar.gz wl@<H200_IP>:/hdd/why/
ssh wl@<H200_IP> "cd /hdd/why/QIHC && tar -xzf ~/offline_bundle_only.tar.gz"
```

## 阶段 3：离线 H200（按顺序执行）

默认路径示例：`/hdd/why/QIHC`（按实际修改）。

### Step 3.1 环境检查

```bash
cd /hdd/why/QIHC

nvidia-smi -L
# 应看到 4 张 H200

ls offline_bundle/manifest.env
ls offline_bundle/models/Qwen2.5-7B-Instruct/config.json
```

### Step 3.2 拉代码（若 H200 能访问内网 Git 服务器）

```bash
git pull origin main
chmod +x experiments/nsfc_evidence/run_offline_h200.sh
```

若完全离线，则从 4090 拷贝的代码包解压即可。

### Step 3.3 离线安装 Python 依赖

```bash
python3 -m venv .venv
source .venv/bin/activate

pip install -U pip
pip install --no-index --find-links=offline_bundle/wheels -e ".[dev,hf,llm]"
```

若 wheels 安装失败：

- 从 4090 **拷贝整个 `.venv`** 到 H200；或
- 请管理员在 H200 上预装 `torch`、`transformers`、`datasets` 等。

### Step 3.4 验证离线环境

```bash
source offline_bundle/manifest.env
export TRANSFORMERS_OFFLINE=1
export HF_HUB_OFFLINE=1
export HF_DATASETS_OFFLINE=1

python -c "import torch; print('GPU count:', torch.cuda.device_count())"
python experiments/download_bbh_hf.py --limit-per-task 5
# 应读取本地缓存，无网络报错
```

### Step 3.5 运行 H200 批量实验（核心一步）

```bash
cd /hdd/why/QIHC
source .venv/bin/activate

# 前台
CUDA_VISIBLE_DEVICES=0,1,2,3 bash experiments/nsfc_evidence/run_offline_h200.sh

# 推荐：后台（约 4–10 小时）
nohup bash experiments/nsfc_evidence/run_offline_h200.sh \
  > experiments/outputs/nsfc_evidence/offline_h200_launcher.log 2>&1 &

tail -f experiments/outputs/nsfc_evidence/offline_h200_launcher.log
```

**未拷贝 14B 模型时：**

```bash
SKIP_CASE_H=1 bash experiments/nsfc_evidence/run_offline_h200.sh
```

**调整 HF 每任务题数：**

```bash
HF_LIMIT_PER_TASK=25 bash experiments/nsfc_evidence/run_offline_h200.sh
```

### Step 3.6 H200 实验分波安排

| 波次 | GPU 0 | GPU 1 | GPU 2 | GPU 3 |
|------|-------|-------|-------|-------|
| **Wave 1** | bundled 7B 主表（5 seeds） | handoff 密集 σ | HF CR 全量 | Case G 约束 BBH |
| **Wave 2** | Pareto 7B | VCI 消融 Case F | synthetic 200 题 | Max-Cut 标度 |
| **Wave 3** | —（串行 CPU/轻量） | Case B / sampler / F 曲线 / Case H(14B) | | |

### Step 3.7 监控进度

```bash
RUN=$(ls -td experiments/outputs/nsfc_evidence/offline_h200_* | head -1)

tail -f "${RUN}/console.log"
ls "${RUN}"/*.log

watch -n 5 nvidia-smi
```

### Step 3.8 打包结果并拷回

```bash
RUN=$(ls -td experiments/outputs/nsfc_evidence/offline_h200_* | head -1)
ls "${RUN}/offline_h200_results.tar.gz"

# 在 Mac / 4090 上拉取（替换 IP）
scp wl@<H200_IP>:"${RUN}/offline_h200_results.tar.gz" ~/Downloads/
```

## 环境变量速查

### 4090（`run_online_4090.sh`）

| 变量 | 默认 | 说明 |
|------|------|------|
| `CUDA_VISIBLE_DEVICES` | `0` | 使用的 GPU |
| `HF_HOME` | `$PWD/offline_bundle/hf_home` | HuggingFace 缓存 |
| `BUNDLE_ROOT` | `$PWD/offline_bundle` | 离线包根目录 |
| `MODEL_HUB_7B` | `Qwen/Qwen2.5-7B-Instruct` | Hub 模型 ID |
| `DOWNLOAD_14B` | `1` | 是否下载 14B |
| `SKIP_EXPERIMENTS` | `0` | `1` = 只下载不实验 |
| `SKIP_WHEELS` | `0` | `1` = 不打包 wheels |

### H200（`run_offline_h200.sh`）

| 变量 | 默认 | 说明 |
|------|------|------|
| `BUNDLE_ROOT` | `$PWD/offline_bundle` | manifest 所在目录 |
| `CUDA_VISIBLE_DEVICES` | `0,1,2,3` | 四卡 |
| `SEEDS` | `0 1 2 3 4` | unified 消融种子 |
| `HF_LIMIT_PER_TASK` | `50` | HF 每子任务题数 |
| `SKIP_CASE_H` | `0` | `1` = 跳过 7B vs 14B |

离线运行时自动设置：

```bash
TRANSFORMERS_OFFLINE=1
HF_HUB_OFFLINE=1
HF_DATASETS_OFFLINE=1
```

## 常见问题

| 问题 | 处理 |
|------|------|
| H200 报 `Connection error` / 无法下载模型 | 确认已 `source offline_bundle/manifest.env`；`MODEL_LOCAL_7B` 指向本地 `config.json` |
| H200 `pip install` 失败 | 使用 `offline_bundle/wheels`；或从 4090 拷贝 `.venv` |
| 4090 OOM | 减小 `LLMSamplerConfig.batch_size`；或 `SKIP_EXPERIMENTS=1` 只传数据 |
| 未拷 14B | H200 设置 `SKIP_CASE_H=1` |
| 某 Wave 子任务失败 | 查看 `offline_h200_*/w1_*.log`，单独重跑对应 Python 命令 |
| H200 不能 `git pull` | 从 4090 用 tar/scp 同步整个 `QIHC/`（排除 `.venv`） |

```bash
cd /path/to/QIHC
git pull origin main
chmod +x experiments/nsfc_evidence/run_dual_4090.sh

# 前台
CUDA_VISIBLE_DEVICES=0,1 bash experiments/nsfc_evidence/run_dual_4090.sh

# 后台（推荐，约 12–24 小时，视 HF_LIMIT 而定）
nohup bash experiments/nsfc_evidence/run_dual_4090.sh \
  > experiments/outputs/nsfc_evidence/dual_4090_launcher.log 2>&1 &
```

```bash
HF_LIMIT_PER_TASK=25 SEEDS="0 1 2" bash experiments/nsfc_evidence/run_dual_4090.sh
```

|------|---------|--------|

| handoff / Pareto / Case G | ✅ | ✅ |

| 预计总时长 | 12–48 h | 4–10 h |

### 清华源 / HF 镜像（默认已开启）

实验脚本会自动加载 `experiments/nsfc_evidence/cn_mirror_env.sh`：

- **PyPI**：`https://pypi.tuna.tsinghua.edu.cn/simple`
- **HuggingFace**：`https://hf-mirror.com`（`HF_ENDPOINT`）
- **禁用 Xet**：`HF_HUB_DISABLE_XET=1`（避免 `cas-server.xethub.hf.co` 401）

若模型下载仍失败，单独执行：

```bash
source experiments/nsfc_evidence/cn_mirror_env.sh
python experiments/nsfc_evidence/download_model_hf.py \
  --repo Qwen/Qwen2.5-7B-Instruct \
  --cache-dir "$HF_HOME"
```

恢复官方 PyPI：`USE_CN_MIRROR=0 bash experiments/nsfc_evidence/run_dual_4090.sh`
